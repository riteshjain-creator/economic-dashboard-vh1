Loop B — Backtest deterministic · 1 run
Objective
The council model's regime calls directionally match actual Nifty weekly returns > 65% of the time across 26 weeks (Dec 2025 – Jun 2026), validating the expert framework before daily production use.
Metric 1
Regime accuracy — for each backtest week, compare india_regime call to actual Nifty 5-day return: Risk-On correct if return ≥ 0%; Risk-Off correct if return ≤ −1.5%; Neutral correct if |return| ≤ 1.5%. Pass: accuracy > 65%.
Metric 2
Invalidation hit rate — % of weeks where model flags invalidation_risk high/medium AND a thread-invalidating event is present in that week's digest entry. Pass: hit rate > 50%.
On FAIL
Surface top-3 missed weeks with rationale → propose targeted council.md edits → re-run. Max 2 fix iterations.
Prerequisite
≥12 approved events from the Backtest Events tab seeded into history-digest.jsonl before backtest.py runs.
What Explainability means here: The dashboard doesn't just report WHAT moved — it explains WHY. A PM reading the POV should be able to trace every regime call to a specific causal chain (e.g. "DXY broke 101 → I1 model fired → FII inflow signal → Fed Pivot thread Heating"). These 4 structural checks verify the pipeline enables that causal chain — without needing to run the Claude API.
Explainability Checks — L1 (Data Pipeline) L1
E1-L1a
TRENDS block in call_claude() prompt contains directional data for ≥4 series (Nifty, India VIX, Brent, US 10Y). Each series provides a 90-day directional signal that Claude can use to explain indicator moves.
Fail if: fewer than 4 series in TRENDS block, or all series show "flat" due to chart_data being empty.
E1-L1b
Historical context block fires when history-digest.jsonl has ≥1 entry. The load_digest_context() function returns a non-empty string, and the digest_block variable is injected into the user prompt.
Fail if: digest block is always empty regardless of history-digest.jsonl state (code path never fires).
E1-L1c
Synthesis instruction in council.md explicitly mandates causal language: contains "explain WHY" AND "causally" AND "name the specific event that inflected each 90-day trend" (when historical context present).
Fail if: instruction only says "explain direction" without causal connector language.
Explainability Checks — L2 (Model Traceability) L2
E2-L2a
Every T/I model (8 total) has ≥1 numeric threshold in its definition. Model signals are threshold-labeled, not vague: "PCR < 0.70" not "low PCR"; "DXY > 104" not "high DXY".
Fail if: any model uses qualitative language without a number anchor.
E2-L2b
Each investment model (I1–I4) is labeled with its thread in the prompt (e.g. "I1 DXY Regime *(Thread: Fed Pivot)*"). When the model fires, Claude can directly name which thread it affects and how.
Fail if: models are unlabeled or thread mapping is absent from prompt.
